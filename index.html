Состояния интерактивных элементов

Состояния интерактивных элементов

Интерактивные элементы — это элементы, с которыми пользователи могут взаимодействовать

Состояния интерактивных элементов

  • normal — элемент интерактивен и включен
  • hover — пользователь навел курсор на элемент
  • focus — пользователь выделил элемент, используя клавиатуру или другой метод ввода
  • active — активное или нажатое состояние
  • progress/loading — элемент находится в процессе завершения действия
  • disabled — элемент в настоящее время не является интерактивным, может быть включен в будущем

Состояния интерактивных элементов

  1. :hover
  2. :focus
  3. :active
  4. :disabled
  5. :focus-visible

Состояния интерактивных элементов

Псевдоклассы в CSS — это специальные селекторы, которые уточняют состояние или тип элемента 

        a:hover {
          color: red;

Основные состояния интерактивных элементов

Состояние :hover срабатывает, когда пользователь наводит курсор на элемент

        a:hover {
          color: red;
        }

Основные состояния интерактивных элементов

Состояние :focus срабатывает при клике на элемент или переходе на него с клавиатуры при помощи клавиши tab

        a:focus {
          color: orange;
        }

Основные состояния интерактивных элементов

Состояние :active срабатывает, когда пользователь взаимодействует с элементом, например, когда удерживает кнопку мыши на элементе

        a:active {
          color: black;
        }

Состояния и доступность сайта

Не все пользователи используют мышь, поэтому важно, чтобы по сайту можно было перемещаться при помощи клавиатуры или других альтернативных устройств

Анимации в CSS

@keyframes

Анимация — это переход от одного состояния элемента к другому состоянию

Чтобы рассказать браузеру, с чего начать и чем закончить анимацию, используется директива @keyframes

Анимации в CSS

        @keyframes circle-to-square {
          from {
            width: 50px;
            height: 50px;
            background-color: #4d58f0;
          }
          to {
            width: 200px;
            height: 200px;
            background-color: #309b30;
          }
        }

Анимации в CSS

        @keyframes circle-to-square {
          from {
            width: 50px;
            height: 50px;
            background-color: #4d58f0;
          }
          50% {
            width: 50px;
            height: 200px;
            background-color: #82309b;
          }
          to {
            width: 200px;
            height: 200px;
            background-color: #309b30;
          }
        }

Задает имя анимации и присваивает ее элементу, к которому она применяется

        }

animation-duration

Задает длительность одного цикла анимации

        .circle {

animation-iteration-count

Задает количество повторений анимации

animation-direction

Сообщает браузеру, должна ли анимация проигрываться в обратном порядке

  1. normal — значение по умолчанию, анимация воспроизводится от начала до конца, после чего возвращается к начальному кадру
  2. reverse — анимация проигрывается в обратном порядке, от последнего ключевого кадра до первого, после чего возвращается к последнему кадру
  3. alternate — каждый нечётный повтор (первый, третий, пятый) анимации воспроизводится в прямом порядке, а каждый чётный повтор (второй, четвёртый, шестой) анимации воспроизводится в обратном порядке
  4. alternate-reverse — аналогично значению alternate, но чётные и нечётные повторы меняются местами

animation-direction

Сообщает браузеру, должна ли анимация проигрываться в обратном порядке

      .circle {
        animation-name: circle-to-square;
        animation-duration: 5s;
        animation-iteration-count: infinite;
        animation-direction: alternate;
      }

animation-direction

Сообщает браузеру, должна ли анимация проигрываться в обратном порядке

animation-timing-function

Устанавливает, согласно какой функции времени должна происходить анимация каждого цикла между ключевыми кадрами (Demo)

  1. ease — значение по умолчанию. Анимация начинается медленно, затем быстро разгоняется и снова замедляется к концу
  2. ease-in — начинается медленно и плавно ускоряется к концу
  3. ease-out — начинается быстро и плавно замедляется к концу
  4. ease-in-out — начинается и заканчивается медленно, ускоряясь в середине
  5. cubic-bezier(x1, y1, x2, y2) — временная функция, описывающая тип ускорения в виде кривой Безье
  6. steps(количество шагов, положение шага) — анимация воспроизводится шагами, резко переходя от одного состояния к другому

animation-timing-function

      .circle {
        animation-name: circle-to-square;
        animation-duration: 5s;
        animation-iteration-count: infinite;
        animation-direction: alternate;
        animation-timing-function: ease-in;
      }

animation-play-state

Свойство, позволяющее ставить анимацию на паузу и запускать снова

  1. running — анимация проигрывается (значение по умолчанию)
  2. paused — анимация ставится на паузу
      .circle:hover, 
      .square:hover {
        animation-play-state: paused;
      }

animation-play-state

Наведите курсор на фигуру,чтобы остановить анимацию

animation-fill-mode

Сообщает браузеру, нужно ли применять стили ключевых кадров до или после проигрывания анимации

  1. none — стили анимации не применяются до и после проигрывания анимации (значение по умолчанию)
  2. forwards — после окончания анимации элемент сохранит стили последнего ключевого кадра
  3. backwards — после окончания анимации к элементу будут применены стили первого ключевого кадра
  4. both — до начала анимации к элементу применяется первый ключевой кадр, а после окончания анимации элемент останется в состоянии последнего ключевого кадра

animation-fill-mode 

      animation-fill-mode: forwards;

transform

transform позволяет манипулировать элементами, изменяя их форму, размер и положение

  1. none — никакого преобразования не применяется
  2. rotate(angle) — вращает элемент на заданный угол
  3. scale(x, y) — масштабирует элемент по осям X и Y
  4. translate(x, y) — перемещает элемент на заданные координаты
  5. skew(x, y) — наклоняет элемент на заданные углы по осям X и Y

Поток документа

  1. блочные элементы занимают всю ширину документа. Располагаются вертикально
  2. строчные элементы не переносятся на новые строки; строчные элементы располагаются на той же строке, что и другие и занимают всю ширину родительского элемента, до тех пор, пока не закончится пространство

Координатная сетка

Система координат в браузере аналогична плоской системе координат из школьной алгебры

Координатная сетка

Выделяют две системы координат:

  1. относительно окна браузера. Например, position: fixed, отсчёт идёт от верхнего левого угла окна
  2. относительно документа. Например, position: absolute на уровне документа, отсчёт идёт от верхнего левого угла документа

top, bottom, left, и right

top, bottom, left, и right используются вместе со свойством position чтобы указать куда именно перемещать позиционируемый элемент. Значения координат могут быть и отрицательными

Ретина-дисплеи

Ретиной называют дисплеи Apple с повышенной плотностью пикселей на квадратный дюйм

Как узнать плотность пикселей

  • Apple: зайти на официальный сайт, выбрать интересующий девайс и нажать на кнопку «Спецификация»
  • для других устройств: можно воспользоваться онлайн-калькулятором (7pads.com, myresolutionis.ru или prostudio.ru)

Особенности вёрстки под ретина-экраны

  1. текст: все тексты на сайте нужно добавлять не изображениями, а с помощью <p>, <h1...h6>, <span> и других тегов. Исключение — логотипы
  2. иконки и декоративные изображения: лучшее решение для иконок и декоративных элементов — использование формата SVG
  3. контентные и фоновые изображения: обычно используют PNG, JPG, WebP и другие растровые форматы

Особенности вёрстки под ретина-экраны

Графика должна быть подготовлена как минимум для двух вариантов экранов. Один для обычных дисплеев, второй, с увеличенным количеством пикселей — для ретины
